## 1Ô∏è‚É£ Docker Environment & System
### Check Docker Version
 $ docker --version
* Confirms Docker installation
* Useful for compatibility checks

### Docker Help
 $ docker help
 $ docker <object> --help
* Lists all commands and options
* Works for `image`, `container`, `volume`, `network`


### Docker System Info
 $ docker info
* Engine configuration
* Containers (running / stopped / paused)
* Images, storage driver, root directory



## 2Ô∏è‚É£ Image Management (Core)
### Search Images (Docker Hub)
 $ docker search <image>
Essential fields to check:
* ‚≠ê Stars (popularity)
* ‚úÖ Official (trusted images)


### Pull Images
 $ docker image pull <image>
docker image pull <image>:<tag>
Essential options:

* `:latest` ‚Üí default tag
* Specific tags ensure reproducibility

---

### List Images
 $ docker image ls
Options:

* `-a` ‚Üí show all images (including intermediate layers)
* `--digests` ‚Üí show image digests

---

### Remove Images
 $ docker image rm <image>
Options:
* `-f` ‚Üí force removal (even if referenced)

---

## 3Ô∏è‚É£ Container Lifecycle (Very Important)
### Run a Container
 $ docker container run <image>
Essential options:

* `-d` ‚Üí detached (background)
* `-it` ‚Üí interactive terminal
* `--name <name>` ‚Üí custom container name
* `-p hostPort:containerPort` ‚Üí port mapping
* `--rm` ‚Üí auto-remove container on stop

---


### List Containers
 $ docker container ls
Options:
* `-a` ‚Üí all containers (running + stopped)
* `-q` ‚Üí container IDs only

---

### Start / Stop / Restart Containers
 $ docker container start <container>
 $ docker container stop <container>
 $ docker container restart <container>
Options:
* `-t <seconds>` ‚Üí timeout before stop

---
### Remove Containers
 $ docker container rm <container>
Options:
* `-f` ‚Üí force remove running container

---

## 4Ô∏è‚É£ Inspecting & Debugging Containers
### View Container Logs
 $ docker container logs <container>
Options:
* `-f` ‚Üí follow logs (stream)
* `--tail <n>` ‚Üí last N lines

---
### Execute Command Inside Running Container
 $ docker container exec -it <container> <command>
Common usage:
 $ docker container exec -it mycontainer /bin/bash


---

### Inspect Container or Image
 $ docker inspect <container|image>
* Outputs detailed JSON metadata
* Useful for networking, mounts, configs

---
